## v1 Requirements ‚Äî All Complete

### Packaging & Distribution

- [x] **PKG-01**: Project is a pip-installable package (src layout, `pyproject.toml`, importable as `forexfactory`) ‚Äî ‚úì Phase 1
- [x] **PKG-02**: A unified CLI entry point exposes populate, refresh, and query commands ‚Äî ‚úì Phase 1
- [x] **PKG-03**: A programmatic library API exposes a main call that returns a path to a parquet file ‚Äî ‚úì Phase 1
- [x] **PKG-04**: Existing scrape/pipeline logic is reused (not rewritten) and the ~195 cached months are re-processed into the new schema without re-scraping ‚Äî ‚úì Phase 1

### Cache

- [x] **CACHE-01**: Data is cached as parquet in a shared user cache dir (default `~/.cache/forexfactory` / OS equivalent), overridable via path/env var ‚Äî ‚úì Phase 1
- [x] **CACHE-02**: Cache scope (currencies, impacts) is chosen when the cache is populated ‚Äî ‚úì Phase 1
- [x] **CACHE-03**: A query that exceeds cached scope auto-fetches the missing data and widens the cache ‚Äî ‚úì Phase 3
- [x] **CACHE-04**: Settled (fully-past) months are never re-fetched automatically ‚Äî manual refresh only ‚Äî ‚úì Phase 1
- [x] **CACHE-05**: Months cached while future-dated are auto-refreshed once the whole month has passed, to fill in `actual` values ‚Äî ‚úì Phase 3
- [x] **CACHE-06**: A force-refresh capability can re-scrape a given range on demand (CLI flag + library arg) ‚Äî ‚úì Phase 3

### Data Schema

- [x] **DATA-01**: Each event record carries the core fields: `datetime_utc`, `currency`, `impact`, `title`, `id`, `leaked` ‚Äî ‚úì Phase 1
- [x] **DATA-02**: Each event carries data values `forecast`, `actual`, `previous`, `revision`, `hasDataValues`, stored as both raw strings and parsed numeric ‚Äî ‚úì Phase 2
- [x] **DATA-03**: Each event carries surprise/identity fields `actualBetterWorse`, `revisionBetterWorse`, `ebaseId`, `country` ‚Äî ‚úì Phase 2
- [x] **DATA-04**: FF UI/internal fields (`checker`, `releaser`, `siteId`, `show*`/`enable*`, `notice`, naming duplicates) are dropped ‚Äî ‚úì Phase 2
- [x] **DATA-05**: Speech/holiday/no-data events are retained in the cache (distinguished by `hasDataValues`); dropping them becomes an optional query-time filter ‚Äî ‚úì Phase 2

### Data Source

- [x] **SRC-01**: The FF JSON/POST endpoint (`apply-settings`) is investigated as a spike; documented decision: NOT ADOPTED ‚Äî HTML `?month=` GET stays bulk primary; `/calendar/more` validated fallback; `/calendar/graph` future enhancement ‚Äî ‚úì Phase 2
- [x] **SRC-02**: HTML-scrape-and-parse-embedded-JS is retained as a fallback source ‚Äî ‚úì Phase 1

### Code Quality

- [x] **QUAL-01**: Deduplication logic is extracted into a single shared helper ‚Äî ‚úì Phase 1
- [x] **QUAL-02**: The `--in-dir` option is honored in all execution paths ‚Äî ‚úì Phase 1

## Out of Scope

| Feature | Reason |
|---------|--------|
| Real-time / streaming calendar updates | Manual refresh + matured-month auto-refresh is sufficient |
| Hosted API server or database backend | This is a local data provider; the parquet cache is the contract |
| Dashboard / GUI | Data provider only; consumers build their own views |
| Storing FF UI/internal fields | No analytical value (see DATA-04) |
| Caching currencies/impacts never queried | Cache stays scoped to what is populated/requested |
