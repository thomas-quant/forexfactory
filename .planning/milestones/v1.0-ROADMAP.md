# Milestone v1.0: Cached Economic Calendar Data Provider

**Status:** ✅ SHIPPED 2026-06-09
**Phases:** 1–3
**Total Plans:** 15

## Overview

Starting from two working loose scripts and ~195 months of raw data on disk, this milestone turns the toolkit into a pip-installable cached data provider. Phase 1 delivered the thin but real end-to-end path: install, populate from existing data, query, get a parquet path back. Phase 2 layered in the full analytical schema (forecast/actual/surprise fields) and investigated the FF API endpoint as a cleaner source. Phase 3 made the cache self-managing: auto-widen on scope miss, auto-mature past-future months, and force-refresh on demand.

## Milestone Summary

**Key Decisions:**

- Distribution model: cached data provider, not bundled dataset or pure tooling lib — fetch once, reuse across projects
- Cache stored as parquet in configurable user cache dir; per-month parquet + `manifest.json` sidecar
- Cache scope configurable at populate time; auto-widen on out-of-scope query (fail-closed via `AutoFetchError`)
- Freshness: manual for settled history; auto-refresh only matured future months (serve-stale-on-failure)
- Schema: core + values (raw+parsed) + FF surprise flags + `ebaseId` for expected-vs-surprise analysis
- Library main call returns parquet path (not DataFrame); consistent with cache contract
- SRC-01 NOT ADOPTED: apply-settings is settings-save only; HTML `?month=` GET stays bulk primary; `/calendar/more` documented as validated fallback
- CR-01 (code review): force-refresh scope-narrowing data-loss bug caught and fixed (union manifest scope before rebuild)

**Issues Resolved:**

- QUAL-01: shared deduplication helper (removed copy-paste between parse and full-pipeline paths)
- QUAL-02: `--in-dir` honored in all execution paths (no silent no-op)
- QUAL-03: failed scrapes no longer write empty JSON files that permanently poison skip logic
- QUAL-04: stale hardcoded date defaults removed
- CR-01: critical data-loss bug where `refresh --force-refresh` at subset scope silently narrowed cached parquets

**Issues Deferred:**

- IN-01: scope-union and settled-check extraction refactor (refactor-only, no behavior change) — Low severity
- IN-02: silent `except pass` in matured-count loop at `_query.py:162-163` — Low severity

**Technical Debt Incurred:**

- DIST-01: Publish to PyPI (deferred to v2; v1 installable from local/git as `-e .`)
- CACHE-V2-01: Chunked/streaming for very wide scopes (deferred to v2)
- SRC-GRAPH-01: `/calendar/graph` numeric per-event time-series — high-value future enhancement
- SRC-MORE-01: `/calendar/more` validated clean-JSON fallback — documented, not adopted as primary
