|----------|-------------|
| Forex Factory server → _scrape parser | Untrusted HTML/JS is fetched over the network and char-walked into JSON by the fragile calendarComponentStates parser |
| network egress → forexfactory.com | Outbound scraping subject to FF terms of service / rate limits |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | DoS / ToS (aggressive scraping) | run_refresh network loop | mitigate | Polite non-zero default delays (1.0s) per D-11; skip-if-cached avoids redundant fetches; tests never hit the live site |
| T-01-06 | Tampering (malformed/hostile HTML) | _scrape.extract_days parser | mitigate | scrape_month catches all exceptions per attempt and returns `[]`; empty result writes nothing (QUAL-03) rather than poisoning the cache |
| T-01-07 | Spoofing detection / IP ban | curl_cffi impersonation + pinned UA | accept | curl_cffi Chrome TLS fingerprint is the primary anti-bot measure; UA pin is a documented secondary risk (CONCERNS) for a personal-use tool |
| T-01-SC | Tampering | dependency installs (curl_cffi/pandas/pyarrow) | accept | No NEW packages introduced — pyproject mirrors the pre-existing, in-use requirements.txt pins; no package-legitimacy gate triggered |
</threat_model>
