<objective>
Build the populate slice — the real cache WRITE half of the walking skeleton. It
ingests the existing on-disk raw `out/` months into one parquet per month under the
cache (D-01), records scope + provenance in the manifest (D-02/CACHE-02/CACHE-04),
and does it with zero network (PKG-04, SC2). Re-runs are incremental and scope-aware
(D-06), and a month whose raw JSON is empty/failed is never permanently skipped
(QUAL-03 / SC5).

Purpose: After this plan a user can `populate` real months into the parquet cache.
Output: `src/forexfactory/_populate.py`, `tests/test_populate.py`.
</objective>
