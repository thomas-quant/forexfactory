<objective>
Deliver the force-refresh vertical slice (CACHE-06): a NEW `--force-refresh` flag on
both `populate` and `refresh`, a mirrored `force_refresh` library kwarg, and the engine
change that bypasses the skip-if-cached logic to re-scrape and overwrite a range on demand.

This is the thinnest end-to-end slice and depends on nothing else in the phase. After it
ships, a user can run `forexfactory populate --force-refresh --start 2025-01` (or
`forexfactory.populate(force_refresh=True, start="2025-01")`) and the specified range is
re-scraped over the network and its parquets overwritten, satisfying SC3.

Honors: D-01 (new flag distinct from existing `--force`), D-02 (refresh gets it too),
D-03 (`force_refresh` kwarg name + `forexfactory.populate()`), D-04 (partial overwrite,
fetched/skipped/failed result dict).

Purpose: Give the cache an on-demand re-scrape escape hatch without touching the
auto-fetch machinery (CACHE-03/05) that later waves add.
Output: `run_refresh`/`run_populate` `force_refresh` kwargs, `forexfactory.populate()`,
CLI `--force-refresh` flags, and tests proving overwrite-on-demand.
</objective>
