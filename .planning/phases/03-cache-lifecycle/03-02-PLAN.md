<objective>
Deliver the matured-month auto-refresh vertical slice (CACHE-05): months scraped while
future-dated (manifest `settled: false`) that have since fully matured are automatically
re-fetched — on both `populate` and `query`/`get()` — so their `actual` values fill in.
A single `auto_fetch=False` knob suppresses it, and a failed re-fetch serves the stale
(forecast-only) cache and warns rather than crashing.

After this slice ships, SC2 holds: a future-dated month re-fetches once it is fully in the
past, with its `actual` values populated, on the next populate or query call.

Honors: D-07 (`auto_fetch=False` reverts to strict cache-only), D-08 (both populate and
query check maturity; all mature months in one call, no cap), D-09 (one knob suppresses
maturity too), D-10 (failure serves stale + warns, deliberately asymmetric vs D-06),
D-11 (CLI prints, library silent), D-12 (matured banner before [N/total] progress).

Depends on 03-01 because the matured re-fetch uses `run_refresh(force_refresh=True)` to
force a re-scrape of the already-cached month.
Purpose: Forecast-only future months mature into expected-vs-surprise data automatically.
Output: `_refresh.refresh_matured_months()`, `auto_fetch`/`session`/`progress` plumbing on
run_query, `auto_fetch`/`session` on run_populate, `auto_fetch` on get()/populate(), the
CLI matured banner, and tests proving each behavior.
</objective>
