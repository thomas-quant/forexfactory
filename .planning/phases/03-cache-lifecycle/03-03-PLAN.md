<objective>
Deliver the scope-miss auto-widen vertical slice (CACHE-03): when a query requests a
currency/impact combination outside the cached scope, instead of erroring the cache
automatically fetches the full configured range for the missing combo, permanently widens
the manifest scope, and returns the matching rows — no manual populate required.

After this final slice ships, SC1 holds: querying EUR/medium when only USD/high is
populated auto-fetches and widens the cache and returns rows.

Honors: D-05 (full-scope widen for the missing combo, not just the query window; permanent
manifest widen), D-06 (failure raises, abort on first failure — deliberately asymmetric vs
CACHE-05's serve-stale), D-07 (`auto_fetch=False` reverts to the existing raise-with-guidance
behavior), D-11/D-12 (CLI scope-miss banner before [N/total] progress).

Depends on 03-02 because it reuses run_query's `auto_fetch`/`session`/`progress` plumbing,
the lazy `_refresh` import pattern, and the cli.py `_print_progress` callback — and because
both plans modify _query.py/cli.py/_refresh.py (sequential, no overlap allowed in one wave).
Purpose: A legitimate out-of-scope request is never blocked; the cache self-widens.
Output: `_exceptions.AutoFetchError`, `_refresh.widen_scope_to_cover()`, the run_query
scope-miss conditional, the CLI scope-miss banner, and tests proving SC1 + the asymmetry.
</objective>
